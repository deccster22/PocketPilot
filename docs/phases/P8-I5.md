# P8-I5 - Monthly / Quarterly Summaries Foundation

## Why This Phase Happened Now

P8-I1 created the first thin Insights history shelf.
P8-I2 added an Insights-specific revisit boundary and calm continuity.
P8-I3 added one deeper interpreted archive shelf.
P8-I4 added one compare-period reflection seam.

That made the next safe step clear:

- keep building on the same interpreted history spine
- add one calm medium-horizon reflection layer
- do it through one service-owned period-summary seam rather than a UI-side recap builder

P8-I5 happens now because PocketPilot already has enough event-history and reflection groundwork to support one honest monthly / quarterly readback, but it is still too early for exports, year-in-review expansion, journaling growth, or analytics theatre.

## What P8-I5 Added

P8-I5 adds five narrow refinements:

1. One canonical `PeriodSummaryVM` contract in `services/insights/types.ts`.
2. One service-owned period-summary builder in `services/insights/createPeriodSummaryVM.ts`.
3. One canonical period-summary fetch seam in `services/insights/fetchPeriodSummaryVM.ts`.
4. One minimal subordinate `InsightsSummaryScreen` that renders prepared period options, summary copy, items, and limitations only.
5. One light navigation handoff from `InsightsScreen` into a monthly / quarterly summaries path.

The read path is now:

`EventLedger -> EventLedgerQueries -> createPeriodSummaryVM -> fetchPeriodSummaryVM -> InsightsSummaryScreen`

## Reflection Rules Locked In Here

P8-I5 establishes these rules:

- one canonical monthly / quarterly summary contract only
- period selection, period shaping, and honest availability stay in `services/`
- `app/` renders prepared summary title, summary, items, and limitations only
- low-change periods are valid and should be stated honestly
- limitations stay sparse and factual
- user-facing summaries stay descriptive, contextual, and non-moralizing
- raw ledger rows, provider/runtime metadata, scorecards, streaks, and shame language stay out of summary copy

## What P8-I5 Deliberately Did Not Add

P8-I5 does not add:

- Year in Review
- export flows
- journal editing
- rich compare-window customization
- scorecards
- streaks
- gamified analytics
- push notifications
- background polling
- AI-generated reflection commentary
- a generic activity report

## What Future P8 Work Can Build On

Later `P8` work can safely extend this seam with:

- richer summary archives that still reuse interpreted history
- Year in Review once it receives dedicated scope
- journal attachment or reflection-note links once the log / journal family is ready
- optional archive-to-summary handoffs if the navigation needs them later

Those later steps should deepen the current interpreted seam rather than replacing it with analytics dashboards, performance theatre, or moralizing recap systems.

## Recommendation Review

- Adopt now: One canonical monthly / quarterly summary seam because the existing P8 history, archive, and compare-period groundwork now supports a calm medium-horizon reflection path without widening scope.
- Add to backlog: Deeper summary archive handoffs and richer period sets because they may help later, but they would broaden P8-I5 beyond a thin foundation.
- Decline for now: Year in Review, exports, journaling expansion, and gamified recap systems because they would turn this phase into a much broader reflection suite.
