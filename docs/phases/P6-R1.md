# P6-R1 — Reorientation Summary / Welcome Back Briefing

## Scope
P6-R1 introduces a service-owned reorientation seam that prepares an optional "welcome back" briefing when a user returns after a meaningful inactivity gap.

## Delivered In This Phase
- explicit reorientation types in `services/orientation`
- a pure deterministic summary builder
- a VM seam that consumes canonical interpreted services
- a lightweight app-facing view seam for rendering prepared output
- tests for thresholds, opt-in behavior, deterministic output, calm language, and item caps
- documentation for product and architecture expectations

## Behaviour Rules
- foreground-only
- no broker APIs
- no order dispatch
- no push notifications
- no background polling
- no inbox/feed expansion
- no guilt language
- no raw signal leakage
- maximum 3 summary items

## Thresholds
- `BEGINNER`: 30 days
- `MIDDLE`: 45 days
- `ADVANCED`: disabled by default unless explicitly enabled

## Canonical Inputs
The reorientation seam must consume interpreted service-owned inputs, not raw UI stitching:
- `EventLedgerQueries`
- `Since Last Checked`
- `OrientationContext`
- `SnapshotModel`

## Acceptance Notes
Availability remains explicit through:
- `NOT_NEEDED`
- `AVAILABLE`

`NOT_NEEDED` reasons remain explicit through:
- `BELOW_THRESHOLD`
- `DISABLED_FOR_PROFILE`
- `NO_MEANINGFUL_CHANGE`
