# P7-K8 - Beginner Inline Glossary / Keyword Help Wiring

## Why This Phase Happened Now

`PX-KI1` refreshed the live glossary-heavy knowledge corpus and `PX-KI2` reconciled taxonomy/catalog references to the family-based `docs/knowledge/<family>/` layout.

With that baseline stable, the next honest runtime step was one narrow inline glossary-help pass that stays optional, calm, and service-owned instead of turning the app into auto-linked wiki clutter.

## What Was Added

- One canonical inline glossary-help contract in `services/knowledge/types.ts`.
- One canonical term selection seam in `services/knowledge/selectInlineGlossaryTerms.ts`.
- One canonical seen-term acknowledgement seam in `services/knowledge/inlineGlossarySeenState.ts`.
- One canonical composition seam in `services/knowledge/createInlineGlossaryHelp.ts`.
- One Dashboard proof path: prepared explanation summary can render linked glossary terms when eligible.
- One Trade Hub proof path: prepared safety copy can render linked glossary terms when eligible.
- Both proof paths route to the existing `KnowledgeTopicScreen` destination.

## What Was Deliberately Not Added

- No execution gating, lockouts, or required-reading mechanics.
- No push, inbox, badge, campaign, or background behavior.
- No whole-app auto-linking sweep.
- No Snapshot glossary rollout.
- No app-owned term matching, ranking, or first-encounter state logic.
- No new explanation screen parallel to the existing topic detail path.

## Future Work This Enables

- Broader but still bounded inline-help rollout to additional explanatory copy once surface fit is explicit.
- Richer profile shaping for middle/advanced users if product evidence supports it.
- Follow-on runtime glossary/help improvements as separate backlog phases, without collapsing ownership boundaries.

## Recommendation Handling

- Adopt now: keep the service-owned inline glossary selection/composition/seen seams and the narrow Dashboard + Trade Hub proof paths.
- Add to backlog: additional surfaces, richer phrasing, and broader glossary affordances only after measured relevance checks.
- Decline for now: auto-link-everywhere behavior, forced tutorials, gating mechanics, push-style coaching, and UI-owned term logic.
