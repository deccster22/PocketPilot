# P7-K4 - Contextual Knowledge Surface Rollout

## Why this phase happened now

P7-K3 proved the contextual-eligibility seam and the Strategy Preview proof path. P7-K4 is the next calm rung because PocketPilot now has enough knowledge foundations to roll that same philosophy into live surfaces without turning it into a feed, a gate, or a second message system.

## What was added

- one canonical contextual-knowledge lane composition seam in `services/knowledge`
- one service-owned prepared lane flowing through Dashboard and Trade Hub
- one calm optional knowledge shelf on each live surface when context is honestly relevant
- one prepared path into the existing Knowledge Topic detail screen from those live surfaces

## What was deliberately not added

- no push notifications
- no background polling
- no forced tutorials
- no execution lockouts
- no gating based on knowledge completion
- no giant recommendation feed
- no app-owned topic selection logic
- no Snapshot clutter

## What future P7 work can build on top of it

- more contextual rollout only when a future surface has honest relevance
- tighter density tuning by profile without changing the service-owned lane contract
- additional calm proof paths only where they stay subordinate to the current surface
- richer knowledge follow-through without moving selection logic into `app/`

## Recommendation handling

- Adopt now: yes
- Add to backlog: small density tuning, any future surface-specific refinements, and additional subordinate proof paths only when justified
- Decline for now: push, inbox, background monitoring, gating, or a recommendation feed
