# P8-I2 - Insights Last-Viewed Continuity

## Why This Phase Happened Now
P8-I1 established the first honest Insights / Event History surface and deliberately stopped before adding deeper archive complexity.

That created the right moment for a narrow next step:
- keep the existing interpreted history seam
- add an explicit Insights-specific last-viewed boundary
- let Insights answer what is meaningfully newer since the user last looked here

P8-I2 happens now because the foundation exists, but the product still needs calm revisit continuity before any broader reflection work should be considered.

## What P8-I2 Added
P8-I2 adds five thin refinements:

1. One canonical Insights continuity contract in `services/insights/types.ts`.
2. One service-owned continuity builder in `services/insights/createInsightsContinuity.ts`.
3. One combined history-plus-continuity fetch seam in `services/insights/fetchInsightsHistoryVM.ts`.
4. One explicit Insights-specific last-viewed read/write path in `services/insights/insightsLastViewed.ts`.
5. One calm continuity note on the top-level Insights surface, still subordinate to Event History itself.

Read path:

`EventLedger -> EventLedgerQueries -> Since Last Checked -> OrientationContext -> createInsightsHistoryVM -> createInsightsContinuity -> fetchInsightsHistoryVM -> InsightsScreen`

Write path:

`InsightsScreen -> markInsightsHistoryViewed -> lastViewedState`

## Continuity Rules Locked In Here
P8-I2 establishes these rules:
- one canonical continuity contract only
- "new" is relative to the Insights surface boundary, not a global app inbox
- continuity counts meaningful interpreted items, not raw event rows
- calm honesty beats novelty padding
- `NO_NEW_HISTORY` is better than fake urgency
- `app/` does not assemble event boundaries or decide what is new

## What P8-I2 Deliberately Did Not Add
P8-I2 does not add:
- compare-period summaries
- archive navigation trees
- journaling
- exports
- reflection scoring
- year-in-review suites
- background polling
- push notifications
- AI-generated reflections
- provider/runtime diagnostics in user-facing Insights
- a raw ledger browser
- unread badges or inbox behavior

## What Future P8 Work Can Build On
Later `P8` work can safely extend this seam with:
- richer reflection summaries when explicitly scheduled
- calmer compare-period framing when it has a dedicated phase
- deeper archive navigation only if the product truly needs it
- more nuanced continuity copy built on the same service-owned boundary

Those later steps should build on the current contract rather than creating a competing history or unread system.

## Recommendation Review
- Adopt now: Persist and advance an explicit Insights-specific last-viewed boundary through services because the first Insights surface now needs revisit continuity without app-side comparison logic.
- Add to backlog: Compare-period or richer reflection summaries because they may add value later, but they would broaden P8-I2 beyond continuity refinement.
- Decline for now: Unread badges, alert counts, or inbox-style novelty because they conflict with PocketPilot's calm interpreted-history posture.
