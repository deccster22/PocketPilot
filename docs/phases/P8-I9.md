# P8-I9 - Journal Subsection Groundwork

## Why This Phase Happened Now

P8-I5 added one calm monthly / quarterly summary foundation.
P8-I6 added one calm summary-archive shelf.
P8-I7 added one calm Year in Review path.
P8-I8 added one calm export-preparation seam over the same reflection family.

That made the next safe step clear:

- keep deepening the same reflection spine instead of starting a separate notes product
- add one canonical journal-entry groundwork seam in `services/`
- let users attach a small optional note to reflection context without turning PocketPilot into a diary ritual, mood tracker, or productivity loop

P8-I9 happens now because the reflection family now has enough calm surfaces to support one honest note attachment path, but it is still too early for reminders, rich editing, exports expansion, analytics on journal content, or AI-generated commentary.

## What P8-I9 Added

P8-I9 adds six narrow refinements:

1. One canonical journal-entry contract family in `services/insights/types.ts` for journal context, prepared entry state, and explicit save / update results.
2. One service-owned journal context resolver and in-memory storage seam in `services/insights/journalShared.ts` and `services/insights/journalEntryStore.ts`.
3. One canonical prepared fetch seam in `services/insights/fetchJournalEntryVM.ts`.
4. One canonical create seam in `services/insights/saveJournalEntry.ts`.
5. One canonical update seam in `services/insights/updateJournalEntry.ts`.
6. One minimal subordinate `InsightsJournalScreen` plus small entry links from the Insights home shelf and the existing summary / Year in Review family when those prepared contexts are available.

The read and write path is now:

`Insights reflection context -> resolveJournalContext -> fetchJournalEntryVM / saveJournalEntry / updateJournalEntry -> InsightsJournalScreen`

## Reflection And Journal Rules Locked In Here

P8-I9 establishes these rules:

- one canonical journal-entry contract only
- one canonical prepared fetch seam only
- journal context resolution stays in `services/`
- journal save / update shaping stays in `services/`
- `app/` renders prepared journal values and submits explicit user text only
- journal notes remain optional, text-only, and small
- journal notes stay context-linked, not free-floating note collections
- user-authored text is never auto-generated, auto-rewritten, or morally scored by the product
- provider/runtime identifiers, raw event metadata, and storage linkage internals stay out of user-facing journal output
- journal copy stays calm, descriptive, and non-gamified

## What P8-I9 Deliberately Did Not Add

P8-I9 does not add:

- a full notes center or second-brain workspace
- reminders, nudges, or push notifications
- background polling
- AI-generated journal commentary
- mood tracking, streaks, scorecards, or performance theatre
- rich text editing, media attachments, or tagging
- journal analytics
- archive browsing for historical journal entries
- journal export expansion

## What Future Reflection Work Can Build On

Later `P8` work can safely extend this seam with:

- broader context-linked journal entry points across additional reflection shelves
- calmer archive access for older journal notes once retrieval rules are explicitly scoped
- explicit log-vs-journal separation if the lighter structured log lane is later introduced
- carefully scoped journal-aware export follow-through only after exports and archive retrieval rules are ready

Those future steps should deepen the same service-owned seam rather than turning Insights into a diary product, coaching engine, or behavior-management loop.

## Recommendation Review

- Adopt now: One canonical journal groundwork layer because the period summary, summary archive, Year in Review, and export-preparation spine now supports one calm optional note path without widening the product into a notes app.
- Add to backlog: Historical journal retrieval, broader context attachment, and journal-aware export follow-through because they may be useful later but widen scope beyond this first grounded seam.
- Decline for now: Reminders, mood scoring, streaks, AI commentary, rich text, media attachments, and analytics on journal content because they would turn reflection into pressure or theatre instead of support.
