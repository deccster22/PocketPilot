# P7-K3 - Knowledge Contextual Eligibility Seam

## Why This Phase Happened Now

`P7-K1` established the first Knowledge shelf.
`P7-K2` established one subordinate topic-detail surface over the canonical normalized knowledge tree.

That baseline is now stable enough for the next thin step:

- not broad contextual rollout
- not a recommendation engine
- not another UI redesign

`P7-K3` happens now because PocketPilot needs one canonical way to answer:

- should a knowledge affordance appear here?
- which topic candidates are most relevant?

without making `app/` guess relevance and without forcing knowledge across the whole product.

## What P7-K3 Added

`P7-K3` adds four connected pieces:

1. One canonical contextual-eligibility contract in `services/knowledge/types.ts`.
2. One service-owned contextual candidate builder in `services/knowledge/createContextualKnowledgeAvailability.ts`.
3. One canonical fetch seam in `services/knowledge/fetchContextualKnowledgeAvailability.ts`.
4. One minimal Strategy Preview proof path that renders a subordinate contextual knowledge affordance only when the seam returns `AVAILABLE`.

The current path is:

```text
knowledgeCatalog
-> createContextualKnowledgeAvailability
-> fetchContextualKnowledgeAvailability
-> Strategy Preview proof path
-> fetchKnowledgeTopicDetailVM
-> Knowledge topic detail screen
```

## Eligibility Groundwork Added

This phase locks in the first small contextual ruleset:

- contextual eligibility is keyed by surface plus interpreted service-owned context
- `services/knowledge` owns candidate selection and ordering
- `app/` receives ordered prepared candidates only
- unsupported or thin context returns explicit `UNAVAILABLE`
- candidate output contains `topicId`, `title`, and a prepared user-facing `reason`
- no raw docs paths, markdown fragments, or source metadata reach the UI

Supported eligibility evaluation surfaces in this phase:

- `DASHBOARD`
- `INSIGHTS`
- `STRATEGY_PREVIEW`

Deliberately still not enabled in this phase:

- `SNAPSHOT`
- `TRADE_HUB`

## Optional And Non-Gating Rules

`P7-K3` preserves the existing PocketPilot posture:

- contextual knowledge remains optional
- unavailable knowledge never blocks another flow
- the Strategy Preview proof path stays subordinate and easy to ignore
- there is no forced reading, unlock, badge, or progression mechanic
- no background work, polling, or push delivery is introduced

## What P7-K3 Deliberately Did Not Add

`P7-K3` does not add:

- contextual rollout across every major surface
- contextual knowledge UI on Dashboard, Snapshot, Trade Hub, and Insights all at once
- a recommendation engine
- AI-generated topic summaries
- raw docs browsing or markdown rendering in app
- quizzes, streaks, or progress mechanics
- push notifications or background polling
- trading execution

## Why The Phase Is Narrow

Contextual knowledge becomes dangerous quickly when eligibility rules are vague.

That is why `P7-K3` prefers:

- one canonical contract over ad hoc surface logic
- one service-owned fetch seam over UI heuristics
- a few strong candidates over noisy coverage
- honest `UNAVAILABLE` results over filler
- one Strategy Preview proof path over broad rollout

## What Future P7 Work Can Build On

Later `P7` work can build on this seam by:

- enabling additional surfaces one at a time through the same fetch seam
- enriching candidate mappings once the canonical knowledge tree carries stronger contextual metadata
- widening contextual topic detail access without duplicating knowledge paths
- layering in smarter ordering only after the current deterministic seam has proven useful

Those later steps should extend the same `services/knowledge` seam instead of inventing parallel relevance logic in `app/`.

## Recommendation Review

- Adopt now: one canonical contextual-eligibility seam with service-owned candidate ordering and one narrow Strategy Preview proof path.
- Add to backlog: richer catalog-level strategy, event, and signal metadata so later contextual matching can rely less on small curated maps.
- Decline for now: broad cross-app contextual rollout, recommendation scoring, and AI-generated knowledge summaries.
