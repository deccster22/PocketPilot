# P9-S6 - Strategy Navigator Fit Contrast ("Why This, Not That")

## Why This Phase Happened Now

`P9-S1` through `P9-S5` already established a stable Strategy Navigator lane:

- one finite simulated preview contract
- one service-owned explanation seam
- one service-owned scenario-contrast seam
- one optional knowledge follow-through seam
- one compact, calm rendering hierarchy

That made the next honest step clear:

- keep the same lane
- keep the same calm posture
- answer one practical comparative question more directly

`P9-S6` happens now to add one service-owned fit-contrast seam that can explain why the current strategy fit reads stronger than nearby alternatives in the present simulated context, without drifting into recommendation theatre, predictions, or execution prompts.

## What P9-S6 Added

P9-S6 adds four narrow pieces:

1. One canonical strategy fit-contrast contract in `services/strategyNavigator/types.ts`.
2. One canonical service-owned builder in `services/strategyNavigator/createStrategyFitContrast.ts`.
3. One Strategy Navigator VM integration path that now carries preview, explanation, scenario contrast, fit contrast, and knowledge follow-through through the same service-owned seam.
4. One subordinate Strategy Navigator proof path in app rendering that displays prepared fit-contrast output only.

The path is:

`selected strategy + finite scenario traits + prepared preview focus + nearby strategy options -> createStrategyFitContrast -> createStrategyNavigatorVM -> fetchStrategyNavigatorVM -> createStrategyNavigatorScreenViewData -> StrategyPreviewCard`

## What P9-S6 Preserved

P9-S6 keeps these behaviors intact:

- existing preview focus contract
- existing explanation wording path
- existing scenario-contrast wording path
- existing optional knowledge follow-through path and topic routing
- same passive Strategy Navigator surface posture
- same deterministic behavior once inputs are supplied

No execution path, dispatch path, or hidden automation was introduced.

## Fit-Contrast Rules Locked Here

P9-S6 establishes these rules:

- one canonical strategy fit-contrast contract
- one canonical service-owned contrast builder
- app renders prepared fit-contrast output only
- fit-contrast stays comparative, compact, and interpretation-first
- one selected strategy is treated as the current fit anchor, compared with one or two nearby alternatives
- ambiguity remains visible when context is mixed or unresolved
- unavailable is honest (`NO_COMPARABLE_CONTEXT`) when service context is not sufficient
- no ranking ladders, scoreboards, or winner language
- no recommendation, prediction, or execution language

## What P9-S6 Deliberately Did Not Add

P9-S6 does not add:

- trade execution prompts
- strategy scoreboards or top-N ranking views
- confidence scoring theater
- push/inbox/badge mechanics
- new background work
- a second strategy engine
- broad redesign of Snapshot, Dashboard, Trade Hub, or Knowledge Library

## What Future Strategy Navigator Work Can Build On

Later `P9` work can build on this seam with:

- richer nearby-alternative selection heuristics when there is stronger context support
- deeper contrast nuance when more scenario families are intentionally added
- stronger cross-linking from fit-contrast output into existing knowledge topics, if still optional and non-gating

Future work should continue to deepen this same service-owned lane rather than creating parallel app-owned ranking or recommendation systems.

## Recommendation Review

- Adopt now: One canonical fit-contrast seam and one calm "why this, not that" proof path, because the current Strategy Navigator stack is mature enough to support comparative clarity without widening product promise.
- Add to backlog: Modest deepening of nearby-alternative selection quality and optional fit-contrast-to-knowledge continuity, only after this seam proves stable.
- Decline for now: Prediction framing, strategy leaderboards, and execution-oriented next-step mechanics, because they would shift Strategy Navigator away from its interpretation-first doctrine.
