# P7-K9 - Glossary Alias / Matching Normalization

## Why This Phase Happened Now

`P7-K8` established the first runtime inline glossary-help seams and kept the rollout intentionally narrow on Dashboard explanation and Trade Hub safety copy.

With that baseline stable, the next honest step was to improve service-owned term resolution quality so canonical terms and approved aliases match more reliably without widening surfaces or moving matching logic into `app/`.

## What Was Improved

- Added one canonical glossary alias/index seam in `services/knowledge/createGlossaryTermIndex.ts`.
- Added explicit glossary variant types in `services/knowledge/types.ts` for canonical term plus match-term variants.
- Refined `services/knowledge/selectInlineGlossaryTerms.ts` to consume the new index seam and match deterministically with normalization for:
  - case variants
  - punctuation/hyphen variants
  - compact variants such as `MarketEvent` vs `market event`
- Tightened noisy matching by removing generic over-link behavior (for example, generic `confirmation` matching on Trade Hub safety copy).
- Kept seen-term acknowledgement behavior service-owned and intact while alias variants still resolve to the same topic-level acknowledgement scope.

## What Was Deliberately Not Added

- No new runtime glossary surfaces beyond existing K8 proof paths.
- No whole-app auto-linking.
- No app-owned term matching, alias expansion, or ranking logic.
- No push, inbox, badge, feed, or background behavior.
- No execution gating, lockouts, or forced-reading behavior.
- No glossary corpus rewrite or external content import.

## Future Work This Enables

- Broader but still bounded alias coverage sourced from stabilized glossary metadata as additional terms prove useful.
- Additional narrow surface rollouts only after relevance and noise controls remain explicit.
- Future glossary/help upgrades without changing the current ownership boundary (`services/knowledge` prepares, `app/` renders).

## Recommendation Handling

- Adopt now: keep the canonical alias/index seam and refined deterministic selector.
- Add to backlog: broader alias coverage and additional surfaces only after bounded relevance validation.
- Decline for now: whole-app auto-linking, UI-owned matching, and high-noise tooltip/wiki behavior.
