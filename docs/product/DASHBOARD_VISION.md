# Dashboard Vision

Dashboard should answer a single calm question: what matters most right now across the user's portfolio?

Unlike Snapshot, Dashboard is not a three-field orientation card. It is a structured prioritised surface that helps the user notice important interpreted changes across symbols, accounts, and strategies without exposing raw signal internals or creating pressure.

For P4-3 the product contract is:
- Dashboard consumes prepared service models, not raw event lists in React
- `prime`, `secondary`, and `background` communicate importance without hype
- profile shaping reduces density for beginners and preserves breadth for advanced users
- prioritisation stays transparent and deterministic
- estimated data stays explicitly labeled and never appears confirmed

Dashboard intentionally excludes charts, journaling, notification logic, analytics scoring, and final layout decisions in this phase.
