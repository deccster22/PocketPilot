# P9-S1 - Strategy Navigator / Preview Foundation

## Why This Phase Happened Now
PocketPilot already had the core ingredients needed for a first Strategy Navigator rung:

- the Strategy Catalog in `P1`
- interpreted MarketEvent, Snapshot, and Dashboard seams from `P3` and `P4`
- calm message and explanation groundwork from `P6`, `PX-C1`, `PX-C2`, `PX-E1`, and `PX-E2`
- top-level educational and reflective surfaces from `P7` and `P8`

That makes this the first safe moment to add a thin exploratory strategy surface without falling into live-simulation pressure, raw signal leakage, or execution-path coupling.

P9-S1 happens now because PocketPilot can finally show how different strategies change interpretation while still keeping the product simulated, descriptive, and non-directive.

## What P9-S1 Added
P9-S1 adds six narrow pieces:

1. One canonical `StrategyNavigatorVM` family in `services/strategyNavigator/types.ts`.
2. One finite simulated scenario catalog in `services/strategyNavigator/strategyPreviewScenarios.ts`.
3. One service-owned preview builder in `services/strategyNavigator/createStrategyNavigatorVM.ts`.
4. One canonical fetch seam in `services/strategyNavigator/fetchStrategyNavigatorVM.ts`.
5. One minimal top-level `StrategyNavigatorScreen` plus `strategyNavigatorScreenView.ts` and `StrategyPreviewCard.tsx`.
6. Documentation and tests that lock the preview into a simulated, descriptive posture.

The current path is:

`core strategy catalog -> strategyPreviewScenarios -> createStrategyNavigatorVM -> fetchStrategyNavigatorVM -> StrategyNavigatorScreen`

## Descriptive Preview Rules Locked In Here
P9-S1 establishes these rules:

- one canonical preview contract only
- strategy and scenario selection remain finite and deterministic
- services own scenario shaping and interpretation wording
- app renders prepared contracts only
- preview output explains interpretation differences, not outcomes
- unavailable is better than filler
- no execution, broker, or live-market semantics
- no raw signal, provider, or runtime leakage
- no urgency language

## What P9-S1 Deliberately Did Not Add
P9-S1 does not add:

- live market simulation
- execution handoff
- broker integration
- order dispatch
- prediction scoring
- AI commentary
- scenario editing
- full onboarding integration
- broad Knowledge Library contextual rollout
- a full interface-transformation sandbox

## What Future P9 Work Can Build On
Later `P9` work can extend this seam with:

- richer strategy-specific transformation of existing surfaces
- deeper knowledge links that still reuse the same prepared preview contract
- more scenario coverage once the starter catalog proves useful
- additional navigational depth only when the product needs it

Those later steps should deepen the current service-owned preview lane instead of replacing it with a simulator or advice engine.

## Recommendation Review
- Adopt now: One canonical Strategy Navigator VM plus one finite scenario catalog, because the existing Snapshot, Dashboard, and explanation groundwork now supports a calm contrast surface without widening into live simulation.
- Add to backlog: Strategy-linked knowledge follow-through from the preview card, because it would add educational value later but is not required to prove the first preview seam.
- Decline for now: Full surface transformation, onboarding orchestration, and freeform scenario tooling, because they would turn this phase into a much larger P9 program rather than a first foundation.
