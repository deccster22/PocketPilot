# P8-I1 - Insights / Event History Foundation

## Why This Phase Happened Now
PocketPilot already had the enabling seams for a first honest reflection surface:
- `MarketEvent` as the canonical interpreted event
- `EventLedger` as persistent interpreted history
- `EventLedgerQueries` as deterministic retrieval
- `Since Last Checked` as the first recency boundary
- `OrientationContext` as prepared history assembly above those seams

P8-I1 happens now because those seams are stable enough to support one thin product-facing history surface without inventing a giant archive system, UI-owned event assembly, or reflective theatre.

## What P8-I1 Added
P8-I1 adds five foundation pieces:

1. One canonical `InsightsHistoryVM` contract in `services/insights/types.ts`.
2. One service-owned history builder in `services/insights/createInsightsHistoryVM.ts`.
3. One canonical fetch seam in `services/insights/fetchInsightsHistoryVM.ts`.
4. One minimal top-level `InsightsScreen` in `app/screens/InsightsScreen.tsx`.
5. One small presentation path in `app/` that renders prepared sections and entries only.

The first product path is:

`EventLedger -> EventLedgerQueries -> Since Last Checked -> OrientationContext -> createInsightsHistoryVM -> fetchInsightsHistoryVM -> InsightsScreen`

## Meaningful-History Rules Locked In Here
P8-I1 establishes the first Event History rules:
- history is interpreted, not raw
- repetitive event rows compress toward fewer stronger notes
- titles and summaries stay calm, factual, and non-moralizing
- absent is better than filler when history is too thin
- `app/` does not group, filter, or narrate ledger rows locally

## What P8-I1 Deliberately Did Not Add
P8-I1 does not add:
- journaling
- exports
- compare-period summaries
- year-in-review systems
- behaviour scoring
- push notifications
- background polling
- AI reflections
- provider/runtime diagnostics in user-facing history
- a generic archive browser
- a second competing history path in `app/`

## Why The First Surface Is Top-Level Insights
The first consumer is a top-level Insights/Event History screen because it gives `P8` one honest home without disturbing existing surface roles:
- Snapshot stays current-state first
- Dashboard stays focus/explanation first
- Trade Hub stays action-boundary first
- Insights becomes the first optional reflective shelf

That keeps reflection additive rather than sprayed across the product.

## What Future P8 Work Can Build On
Later `P8` work can safely extend this seam with:
- richer time windows and last-view handling
- broader reflection summaries
- calmer compare-period framing
- eventual journaling or export layers when they are explicitly scheduled

Those later steps should build on the current service-owned contract rather than creating competing history builders.
