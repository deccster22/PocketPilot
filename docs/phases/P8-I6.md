# P8-I6 - Summary Archive Foundation

## Why This Phase Happened Now

P8-I1 created the first thin Insights history shelf.
P8-I2 added calm last-viewed continuity.
P8-I3 added one deeper interpreted archive shelf for event history.
P8-I4 added one compare-period reflection seam.
P8-I5 added one monthly / quarterly summary foundation.

That made the next safe step clear:

- keep building on the same interpreted history spine
- give monthly / quarterly summaries one calm place to be revisited later
- do it through one service-owned archive seam rather than UI-side archive shaping

P8-I6 happens now because PocketPilot already has enough summary groundwork to support one honest archive shelf, but it is still too early for Year in Review, exports, journaling expansion, or a broad reporting center.

## What P8-I6 Added

P8-I6 adds five narrow refinements:

1. One canonical `SummaryArchiveVM` contract in `services/insights/types.ts`.
2. One service-owned summary-archive builder in `services/insights/createSummaryArchiveVM.ts`.
3. One canonical summary-archive fetch seam in `services/insights/fetchSummaryArchiveVM.ts`.
4. One minimal subordinate `InsightsArchiveScreen` that renders prepared archive cards only.
5. One light handoff from the archive shelf into the existing period-summary detail path.

The read path is now:

`EventLedger -> createPeriodSummaryVM -> createSummaryArchiveVM -> fetchSummaryArchiveVM -> InsightsArchiveScreen -> InsightsSummaryScreen`

## Reflection Rules Locked In Here

P8-I6 establishes these rules:

- one canonical summary-archive contract only
- archive-entry shaping, ordering, and honest availability stay in `services/`
- `app/` renders prepared archive title, summary line, coverage label, and prepared-date label only
- archive entries stay descriptive, contextual, and non-moralizing
- archive cards should feel like a quiet record shelf, not a leaderboard or report card
- raw ledger rows, provider/runtime metadata, scorecards, exports, and shame language stay out of archive output

## What P8-I6 Deliberately Did Not Add

P8-I6 does not add:

- Year in Review
- export flows
- journal editing
- freeform reflection writing
- compare-window customization
- scorecards
- streaks
- gamified analytics
- push notifications
- background polling
- AI-generated reflection commentary
- a generic reporting center

## What Future P8 Work Can Build On

Later `P8` work can safely extend this seam with:

- richer archive shelves that still reuse prepared summary contracts
- stored historical summary versions once persistence scope is ready
- Year in Review once it receives dedicated scope
- journal attachment or reflection-note links once the log / journal family is ready

Those later steps should deepen the current interpreted seam rather than replacing it with analytics theatre, export-heavy recap flows, or moralizing reflection systems.

## Recommendation Review

- Adopt now: One canonical summary-archive seam because P8-I5 already established calm monthly / quarterly summaries and the next honest step is a service-owned shelf for revisiting them.
- Add to backlog: Stored archive versions, richer period sets, and archive-to-journal linkage because they may be useful later, but they widen the reflection family beyond this foundation.
- Decline for now: Year in Review, exports, journaling expansion, and performance-style recap systems because they would turn this phase into a broader reporting suite.
