# P7-K19 - Layer 3 First-Wave Runtime Surfacing

## Why This Phase Happened Now

`P7-K17` standardized the existing Layer 3 market-example/evidence narratives, and `P7-K13` already guards runtime catalog drift for explicit runtime-required topic IDs.  
After that stabilization, the accepted `P7-K18` plan scoped a conservative first runtime wave: surface a small evidence-only subset as follow-through routes before any broad shelf rollout.

## Dependencies

- `P7-K13` for deterministic runtime catalog sync validation.
- `P7-K17` for behavior-first narrative consistency in the example/evidence corpus.
- Accepted `P7-K18` planning recommendation for bounded first-wave runtime scope.

## Why Only Six Evidence Examples Were Surfaced

This phase intentionally surfaces exactly six evidence topics to keep risk bounded while proving runtime follow-through behavior:

- 2 for `strategy-trend-follow`
- 2 for `strategy-breakout-watcher`
- 2 for `strategy-buy-the-dip`

The phase does not attempt broad Layer 3 runtime expansion.

## Why Market Examples Remain Held

`market-examples-*` topics remain docs-side in this wave to avoid over-expanding runtime scope before first-wave follow-through quality is validated.

## Why Trade Hub Example Links Remain Off

Trade Hub linking stays unchanged in this phase to preserve planning-language safety and avoid introducing example-link behavior in action-adjacent surfaces before broader review.

## Hidden Follow-Through Runtime Surfacing Model

First-wave evidence topics are runtime-routable but hidden from top-level Knowledge Library shelves:

- runtime entries exist in `knowledgeCatalog.ts`
- each first-wave evidence entry uses `priority: NEXT`
- `createKnowledgeLibraryVM` continues showing only `priority: NOW` items
- detail routes remain resolvable through `fetchKnowledgeTopicDetailVM`

This delivers follow-through access without a visible Examples shelf.

## What Was Deliberately Not Changed

- no visible Examples shelf
- no market-example runtime routes
- no Trade Hub example related-topic links
- no new example documents
- no placeholder example pages
- no Strategy Navigator/Preview behavior changes
- no strategy engine changes
- no app layout/navigation changes
- no CANON changes
- no `docs/source` changes
