# P9-S7 - Strategy Navigator Nearby-Alternative Selection Heuristics

## Why This Phase Happened Now

`P9-S6` added the first fit-contrast seam for Strategy Navigator.
That seam answered "why this, not that," but nearby-alternative selection was still too static in places.

The next honest rung was not a bigger surface or new recommendation behavior.
The next honest rung was improving comparator quality inside the same service-owned lane.

`P9-S7` happens now to make nearby alternatives more context-adjacent and less arbitrary, while preserving the same calm, non-directive posture.

## What P9-S7 Added

P9-S7 adds four narrow pieces:

1. One canonical nearby-alternative selection contract in `services/strategyNavigator/types.ts`.
2. One canonical service-owned heuristic selector in `services/strategyNavigator/selectNearbyAlternativeStrategies.ts`.
3. One wiring update so `createStrategyNavigatorVM` and `createStrategyFitContrast` consume the same nearby-alternative selection seam.
4. Focused tests that lock deterministic, context-aware comparator selection and preserve non-execution, non-ranking language boundaries.

The path is:

`selected strategy + scenario + prepared focus + strategy metadata -> selectNearbyAlternativeStrategies -> createStrategyFitContrast -> createStrategyNavigatorVM -> fetchStrategyNavigatorVM`

## What P9-S7 Preserved

P9-S7 preserves:

- Strategy Navigator preview behavior
- preview explanation behavior
- preview scenario-contrast behavior
- knowledge follow-through behavior and routing
- subordinate app placement for fit-contrast
- deterministic output once inputs are supplied
- ambiguity visibility in mixed/unsettled scenarios

No execution path, gating behavior, ranking spectacle, or recommendation theater was introduced.

## Nearby-Alternative Heuristic Rules Locked Here

P9-S7 establishes:

- one canonical nearby-alternative availability/selection seam
- service-owned comparator scoring only (no app-owned heuristic logic)
- context-aware selection using stable strategy proximity, scenario weighting, focus cues, and conservative exclusion
- narrow output (1-2 nearby alternatives)
- honest unavailability when no comparable alternatives are supported

## What P9-S7 Deliberately Did Not Add

P9-S7 does not add:

- trade execution prompts
- top-N strategy scoreboards or winner framing
- prediction language or certainty inflation
- push/inbox/badge mechanics
- broader surface rollout outside Strategy Navigator
- knowledge-system rewrites

## What Future Strategy Work Can Build On

Later `P9` work can build on this seam with:

- richer nearby-alternative metadata once more strategy context is intentionally modeled
- stronger exclusion heuristics for edge cases with sparse context
- deeper fit-contrast nuance while keeping calm, interpretation-first wording

Future work should continue deepening the same service-owned lane, not creating app-owned ranking or advice systems.

## Recommendation Review

- Adopt now: Service-owned nearby-alternative heuristics, because they improve comparator relevance inside the existing fit-contrast seam without widening product scope.
- Add to backlog: Additional heuristics and metadata for rarer edge contexts once this first deterministic selector proves stable.
- Decline for now: Strategy leaderboards, confidence scoring, and execution-oriented next-step mechanics, because they break the current product posture.
