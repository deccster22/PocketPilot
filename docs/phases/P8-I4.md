# P8-I4 - Compare-Period / Reflection Summary Foundation

## Why This Phase Happened Now
P8-I1 created the first thin Insights history shelf.
P8-I2 added an Insights-specific revisit boundary and calm continuity.
P8-I3 added one deeper interpreted archive shelf.

That made the next safe step clear:
- keep building on the same interpreted history spine
- compare two meaningful slices in a calm, factual way
- do it through one service-owned seam rather than a UI-side comparison layer

P8-I4 happens now because PocketPilot has enough interpreted history structure to support one honest compare-period note, but it is still too early for journaling, exports, scoring, or broader reflection theatre.

## What P8-I4 Added
P8-I4 adds five narrow refinements:

1. One canonical `ReflectionComparisonVM` contract in `services/insights/types.ts`.
2. One service-owned comparison builder in `services/insights/createReflectionComparisonVM.ts`.
3. One canonical comparison fetch seam in `services/insights/fetchReflectionComparisonVM.ts`.
4. One minimal subordinate `InsightsReflectionScreen` that renders prepared windows, summary items, and limitations only.
5. One light navigation handoff from `InsightsScreen` into a compare-recent-history path.

The compare-period read path is now:

`EventLedger -> EventLedgerQueries -> Since Last Checked -> OrientationContext -> createInsightsHistoryVM -> createReflectionComparisonVM -> fetchReflectionComparisonVM -> InsightsReflectionScreen`

## Interpreted Reflection Rules Locked In Here
P8-I4 establishes these rules:
- one canonical comparison contract only
- comparison windows are selected in `services/`, not `app/`
- comparison copy stays interpretation-first and user-legible
- fewer stronger notes are better than a stats dump
- stable / little-changed outcomes are valid and should be stated honestly
- limitations stay sparse and factual
- raw signal codes, event IDs, strategy IDs, provider/runtime metadata, and scoring language stay out of user-facing reflection

## What P8-I4 Deliberately Did Not Add
P8-I4 does not add:
- journaling
- exports
- year-in-review tooling
- behaviour scoring
- coaching prompts
- push notifications
- background polling
- AI-generated reflections
- provider/runtime diagnostics in user-facing reflection
- a raw ledger comparison browser
- archive search complexity
- a giant analytics dashboard

## What Future P8 Work Can Build On
Later `P8` work can safely extend this seam with:
- additional service-owned comparison modes when the product has a reason for them
- broader reflection summaries that still reuse interpreted history rather than raw logs
- optional archive-to-comparison handoffs if the navigation needs them later
- eventual journaling or export families only when they receive dedicated scope and doctrine

Those later steps should deepen the current interpreted seam rather than replacing it with performance analytics or reflection theatre.

## Recommendation Review
- Adopt now: One canonical compare-period seam because the existing history, continuity, and archive groundwork now supports a calm comparison without broadening scope.
- Add to backlog: Explicit user-selected comparison windows because they may be useful later, but they would widen P8-I4 beyond a thin foundation.
- Decline for now: Journaling, scoring, and export systems because they would turn this phase into a broader reflection suite rather than a first comparison rung.
