# P6-A7 - Since Last Checked Surface Rollout

## Why This Phase Happened Now

P6-A1 established one canonical message-policy seam.
P6-A2 proved the same seam could serve Snapshot, Dashboard, and Trade Hub without becoming a notification platform.
P6-A3 made alert treatment profile-aware.
P6-A4 improved the interpreted inputs that feed the same seam.
P6-A5 added a calm user-visible rationale path on top of that same policy result.
P6-A6 compacted those outputs into one grouped lane so the current consumers could keep reading a single service-owned contract.

The next honest rung was not push, inbox, badges, background polling, or a notification center.
It was one separate calm Snapshot section that could show meaningful change since the last check without turning Snapshot into a feed.

This phase happened now because the event query, history, and last-viewed groundwork were already stable enough to reuse, and because the Snapshot surface was ready for one subordinate section under Strategy Status that stayed compact, calm, and account-scoped.

## What Was Added

- one canonical `SinceLastCheckedAvailability` contract plus explicit item emphasis in `services/orientation/types.ts`
- one canonical service-owned composer in `services/orientation/createSinceLastCheckedVM.ts`
- one canonical service-owned fetch seam in `services/orientation/fetchSinceLastCheckedVM.ts`
- one Snapshot-facing surface field in `services/snapshot/fetchSnapshotSurfaceVM.ts`
- one separate calm Snapshot section under Strategy Status in `app/screens/SnapshotScreen.tsx`
- one presentational section card in `app/components/SinceLastCheckedCard.tsx`
- one prepared Snapshot screen-view seam in `app/screens/snapshotScreenView.ts`
- one alignment update so the existing briefing zone and message-policy inputs can reuse the same prepared Since Last Checked truth
- focused tests proving determinism, capping, empty-state collapse, account-scoped filtering, and passive app rendering

## What Was Deliberately Not Added

- no push notifications
- no unread counts
- no badges
- no notification center
- no background polling
- no hidden automation
- no generic history browser
- no raw event-ledger dump on Snapshot
- no app-owned ranking or filtering logic
- no urgency language or performance theatre

The existing briefing zone remains a separate message-policy lane.
This phase only adds the calm Since Last Checked section under Strategy Status.

## What Future P6 / P8 Work Can Build On

Later `P6` work can build on this phase by:

- tightening calm copy for the same section without changing ownership
- extending the same prepared contract only where a natural inline home already exists
- keeping the message-policy lane and the Since Last Checked section separate but consistent

Later `P8` work can build on this phase by:

- reusing the same history / last-viewed seams for broader reflection surfaces
- layering deeper readbacks on top of the same account-scoped truth
- keeping the product descriptive rather than turning it into a generic archive browser

Future work should still keep PocketPilot foreground-only, service-owned, compact, and non-inbox-like.

## Verification

- added coverage for determinism and max-3 compaction
- added coverage for account-scoped filtering and empty-state collapse
- added coverage for surface threading into Snapshot and passive app rendering
- `npm run verify` remains the merge gate for this phase

## Recommendation Review

### Adopt Now

- Canonical Since Last Checked VM and fetch seam: adopt now because the account-scoped history seam is already stable and the screen needs one prepared contract, not more local logic.
- Separate Snapshot section under Strategy Status: adopt now because it keeps the surface calm and subordinate without widening the product into a feed.
- Passive app rendering of prepared Since Last Checked data: adopt now because the UI should stay a renderer, not a selector.

### Add To Backlog

- Small copy refinements once real usage shows which items need more tightening
- Future reflection-depth work that belongs to the broader `P8` family
- Any later calm inline section only if it has a natural service-owned home

### Decline For Now

- Push notifications
- Inbox, unread, badge, and notification-center mechanics
- Background monitoring or polling
- Generic history browsing on Snapshot
- App-owned event ranking, filtering, or summarization
