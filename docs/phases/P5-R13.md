# P5-R13 - Strategy-Owned Prepared Stop / Target References

## Why This Phase Happened Now

P5-R1 through P5-R12 already stabilized the support-first risk lane: selected basis, risk-per-trade framing, sizing/max-loss output, guidance for thin inputs, optional guardrail preferences/evaluation, and grouped Trade Hub risk composition.

P5-R13 happened now because the grouped lane was stable enough to add one explicit prepared stop/target reference availability seam without widening into execution behavior, settings sprawl, or recommendation theatre.

## What Was Added

- one canonical `PreparedTradeReference` plus `PreparedTradeReferencesAvailability` contract in `services/trade/types.ts`
- one canonical service-owned producer seam: `services/trade/createPreparedTradeReferences.ts`
- explicit unavailable reasons for stop/target publishing:
  - `NO_STRATEGY_REFERENCE`
  - `THIN_CONTEXT`
  - `NOT_ENABLED_FOR_SURFACE`
- deterministic source labeling and limitation notes for strategy-derived references
- plan-level threading via `createProtectionPlans` so selected Trade Hub context carries prepared stop/target availability truth
- preview/session threading so the same prepared availability contract travels through `createTradePlanPreview` and `fetchConfirmationSessionVM`
- risk selector integration in `fetchRiskToolVM` so prepared stop/target references can be consumed through service-owned context when honestly available

## What Was Deliberately Not Added

- no live execution or dispatch behavior
- no broker adapter integration or payload widening
- no multi-target or laddered reward framing
- no account preference surface for stop/target references
- no new settings UX
- no quote-derived predictive exits
- no app-owned stop/target derivation
- no urgency/profit-opportunity framing

## What Future P5 Work Can Build On

- additional strategy-owned prepared stop/target sources when deterministic context remains explicit and honest
- richer non-executional reference annotations if later phases need clearer context explanations
- calm cross-surface reuse of the same availability contract beyond current Trade Hub seams
- future risk-lane expansions that still preserve unavailable-on-thin-context behavior

## Verification

- added direct producer tests for deterministic behavior, available stop/target output, thin-context unavailable states, and no fake-value publishing
- updated plan/preview/session tests so the prepared stop/target availability seam is carried through Trade Hub service contracts
- updated risk-tool fetch tests so selector behavior consumes produced references while preserving user-input precedence
- `npm run verify` remains the merge gate for this phase

## Recommendation Review

### Adopt now

- canonical prepared stop/target availability contract: adopt now because services need one explicit truth shape for honest availability versus thin context
- canonical producer seam in `services/trade/`: adopt now because stop/target publishing should stay deterministic and centralized
- preview/session/risk-tool threading of that seam: adopt now because consumers should reuse one prepared contract without app-side derivation

### Add to backlog

- additional strategy-specific reference sources beyond the current deterministic contexts
- richer explanatory limitation variants when future phases need more context framing
- broader surface reuse outside current Trade Hub seams

### Decline for now

- execution-path widening or order payload behavior
- recommendation or profit-opportunity copy
- multi-target planning expansion
- settings-heavy stop/target preference UX
