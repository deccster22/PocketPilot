# P9-S5 - Strategy Preview Compaction Pass

## Why This Phase Happened Now

`P9-S1` gave Strategy Preview its first calm simulated contract.
`P9-S2` added optional knowledge follow-through on that same service-owned lane.
`P9-S3` added one subordinate explanation shelf.
`P9-S4` added one subordinate scenario-contrast shelf.

That deeper stack is useful, but it makes the preview card easier to overcrowd.
`P9-S5` happens now because the next honest step is not another logic family.
The next honest step is a compaction pass that keeps the same prepared VM readable, calm, and educational as the lane gains depth.

## What P9-S5 Added

P9-S5 adds four narrow things:

1. One render-only compaction helper in `app/screens/strategyNavigatorScreenView.ts` that reshapes the existing prepared preview into a clearer hierarchy for display.
2. One Strategy Preview card refinement in `app/components/StrategyPreviewCard.tsx` that keeps the main preview focus dominant, groups explanation plus contrast into lighter supporting context, and keeps knowledge follow-through clearly optional.
3. Focused app tests that lock stable ordering, clean disappearance of unavailable subordinate sections, and non-directive wording.
4. Documentation updates that explain why this phase compacts the surface instead of widening the logic.

The current path stays:

`createStrategyNavigatorVM -> fetchStrategyNavigatorVM -> createStrategyNavigatorScreenViewData -> StrategyPreviewCard`

The important detail is that `P9-S5` does not widen `services/strategyNavigator/`.
Services still own preview, explanation, contrast, and knowledge shaping.
App still renders prepared contracts only.

## Compaction Rules Locked Here

P9-S5 establishes these rendering rules:

- one dominant main-preview-focus block
- one stable supporting detail order: Dashboard shift, Market events that would matter, Alert posture
- one grouped supporting-context shelf for explanation and contrast when they are honestly available
- one clearly optional helpful-next-reading shelf when knowledge follow-through is honestly available
- unavailable subordinate sections disappear cleanly instead of leaving filler
- no new preview payload family
- no app-side interpretation, scenario logic, explanation logic, or knowledge-selection logic
- no urgency, execution, forecast, or profit framing

## What P9-S5 Deliberately Did Not Add

P9-S5 does not add:

- new scenario families
- new explanation families
- new knowledge recommendation layers
- live simulator behaviour
- execution handoff
- broker integration
- AI commentary
- prediction scoring
- onboarding integration
- a giant accordion or simulator dashboard

## What Future P9 Work Can Build On

Later `P9` work can build on this calmer hierarchy with:

- slightly richer presentation depth if the same compact reading path still holds
- tighter mobile ergonomics if the current shelf grouping proves useful
- deeper preview content only when services truly need a new canonical seam

Those later steps should still prefer one prepared preview lane over multiple competing preview payload families.

## Recommendation Review

- Adopt now: The compact hierarchy and grouped supporting context, because the existing preview is now deep enough to need clearer visual weight without inventing new logic.
- Add to backlog: Small presentational refinements such as more responsive grouping or lighter progressive disclosure if the preview gains one more subordinate shelf later.
- Decline for now: New simulator layers, recommendation engines, or execution-oriented next-step affordances, because they would widen the product promise instead of keeping the preview calm and descriptive.
