# P9-S3 - Strategy Navigator Preview Explanation Deepening

## Why This Phase Happened Now

`P9-S1` established the first calm Strategy Preview contract.
`P9-S2` proved that the same service-owned lane could add optional knowledge follow-through without becoming onboarding or advice.

That makes this the next honest step:

- not a bigger simulator
- not a live explanation engine
- not a recommendation layer

Instead, `P9-S3` adds one thin answer to:

- why does this strategy care about this simulated scenario?

This phase happens now because PocketPilot already has:

- one canonical preview VM
- one finite simulated scenario catalog
- one prepared preview focus shape
- one stable app seam that already renders prepared preview content only

## What P9-S3 Added

P9-S3 adds six narrow pieces:

1. One canonical preview-explanation contract in `services/strategyNavigator/types.ts`.
2. One canonical preview-explanation builder in `services/strategyNavigator/createStrategyPreviewExplanation.ts`.
3. One Strategy Navigator builder path that now shapes preview interpretation, preview explanation, and preview knowledge follow-through together.
4. One canonical fetch seam that still returns a single prepared `StrategyNavigatorVM`.
5. One minimal explanation section on the existing Strategy Preview card when explanation is honestly available.
6. Tests and docs that lock explanation into a calm, descriptive, educational posture.

The current path is:

`selected strategy + selected scenario + prepared preview focus -> createStrategyPreviewExplanation -> createStrategyNavigatorVM -> fetchStrategyNavigatorVM -> Strategy Preview card`

## Descriptive Explanation Rules Locked In Here

P9-S3 establishes these rules:

- one canonical preview-explanation contract only
- services own preview-explanation shaping
- app renders prepared explanation content only
- explanation stays simulated, descriptive, calm, exploratory, and non-directive
- explanation answers worldview and interpretation questions, not outcome questions
- interpreted MarketEvents may appear as prepared explanation language, but raw signal arrays and runtime details stay hidden
- unavailable is better than filler
- no execution, broker, readiness, profit, or forecast semantics
- no urgency language

## What P9-S3 Deliberately Did Not Add

P9-S3 does not add:

- live market simulation
- execution handoff
- broker integration
- order dispatch
- AI commentary
- prediction scoring
- full simulator tooling
- broad explanation rollout across Snapshot, Dashboard, Trade Hub, Knowledge, or Insights
- onboarding integration
- push notifications or background polling

## What Future P9 Work Can Build On

Later `P9` work can extend this seam with:

- richer strategy-specific explanation depth when the current contract proves useful
- broader scenario coverage that still reuses the same canonical explanation contract
- deeper strategy-aware knowledge bridges that stay optional and subordinate
- additional navigator presentation depth that still keeps `services/` in charge of explanation wording

Those later steps should deepen this same service-owned lane instead of replacing it with a simulator, advice engine, or execution funnel.

## Recommendation Review

- Adopt now: One canonical preview-explanation seam, because Strategy Preview now has enough prepared context to explain worldview shifts without drifting into prediction, advice, or simulator scope.
- Add to backlog: Slightly richer explanation depth or broader scenario coverage, because the first version should prove stable before the preview card grows into a larger explanation surface.
- Decline for now: Live simulation, AI strategy commentary, and recommendation-style next-step guidance, because they would break PocketPilot's calm interpretation posture and widen the phase beyond its contract.
