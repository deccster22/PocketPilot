# P9-S2 - Strategy Navigator Preview-to-Knowledge Follow-Through

## Why This Phase Happened Now

`P9-S1` proved that PocketPilot could compare strategies through one calm simulated preview without leaking raw signals or drifting into execution language.

The next honest step is not a larger simulator.
It is a small, service-owned bridge from:

- "how this strategy interprets the simulated scenario"

to:

- "what should I read next if I want to understand this a little better"

This phase happens now because PocketPilot already has:

- the `P9-S1` preview contract
- the `P7` knowledge catalog and topic detail foundation
- enough strategy, scenario, and preview-focus context to choose a few prepared knowledge next steps deterministically

## What P9-S2 Added

P9-S2 adds five narrow pieces:

1. One canonical preview-to-knowledge follow-through contract in `services/strategyNavigator/types.ts`.
2. One canonical preview knowledge selector in `services/strategyNavigator/selectStrategyPreviewKnowledge.ts`.
3. One Strategy Navigator builder path that now shapes preview interpretation and preview knowledge follow-through together.
4. One canonical fetch seam that returns the prepared follow-through on the same `StrategyNavigatorVM`.
5. One minimal preview-card addition that renders optional related reading without changing the surface into onboarding or advice.

The current path is:

`knowledgeCatalog + preview inputs -> selectStrategyPreviewKnowledge -> createStrategyNavigatorVM -> fetchStrategyNavigatorVM -> Strategy Preview card`

## Descriptive Follow-Through Rules Locked In Here

P9-S2 establishes these rules:

- knowledge follow-through stays optional
- services own preview knowledge selection
- app renders prepared items only
- fewer stronger items beat a long noisy list
- preview remains simulated, descriptive, and non-directive even when knowledge is available
- unavailable is better than filler
- no raw docs paths or markdown browsing
- no execution, readiness, or profit semantics

## What P9-S2 Deliberately Did Not Add

P9-S2 does not add:

- live market simulation
- execution handoff
- broker integration
- order dispatch
- AI commentary
- full onboarding integration
- broad Knowledge Library rollout across other surfaces
- a full recommendation engine
- forced reading before deeper preview use

## What Future P9 Work Can Build On

Later `P9` work can extend this seam with:

- richer strategy-specific follow-through once the current selector proves useful
- additional preview presentation depth that still reuses the same prepared VM
- broader strategy-aware explanation shelves when the product has a stronger reason for them
- deeper knowledge integration that still keeps the path optional and non-gatekeeping

Those later steps should deepen the same service-owned preview lane instead of replacing it with a simulator, advice engine, or onboarding funnel.

## Recommendation Review

- Adopt now: One canonical preview-to-knowledge follow-through contract, because Strategy Preview now has enough context to offer a calm next reading step without widening into advice or onboarding.
- Add to backlog: Broader cross-surface follow-through reuse, because the preview-owned seam should prove stable before Strategy Preview logic spreads into Snapshot, Dashboard, or Trade Hub.
- Decline for now: Full recommendation-engine scoring or required learning paths, because they would turn a small optional bridge into supervision theatre and weaken PocketPilot's calm posture.
