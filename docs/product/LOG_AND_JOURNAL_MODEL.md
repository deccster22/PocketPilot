# LOG_AND_JOURNAL_MODEL.md

## Purpose
Defines how PocketPilot handles optional user reflection without becoming a guilt diary or ritual machine.

## Founding stance
Log and Journal should help users remember what mattered, not make them perform self-improvement for the app.

## Distinction
### Log
lighter, quicker, more structured

### Journal
deeper, slower, more reflective

## Product job
- preserve decision context
- support learning memory
- enrich review surfaces
- support reflection without shame

## Non-negotiables
- optional
- non-judgmental
- not required for core product use
- separate from canonical ledger truth

## Drift to block
- streaks
- guilt loops
- forced reflection
- diary coercion
- “good trader / bad trader” scoring
