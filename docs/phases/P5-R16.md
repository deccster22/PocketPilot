# P5-R16 - Trade Hub Plain-Language Surface Copy Cleanup

## Why This Phase Happened Now

`P5-R15` stabilized prepared planning-level terminology and compact stop/target rendering.
After that merge, Trade Hub still exposed internal implementation wording in visible section labels and status copy.

`P5-R16` happens now to keep Beginner and Middle surfaces calm and readable while preserving the same service-owned semantics and non-dispatching boundary.

## What Copy Was Cleaned

- `Prepared surface for <profile>` -> `Planning view for <profile>`
- `Service-owned sizing summary` -> `Prepared sizing summary`
- `Service-owned submission gate` -> `Submission readiness check`
- `Prepared non-dispatching seam` -> `Non-submitting safety check`
- `Simulated-only adapter seam` -> `Simulated adapter check`
- execution-boundary section labels now prefer plain-language wording:
  - `Execution preview`
  - `Submission readiness`
  - `Submission check`
  - `Execution handoff`
- one explicit boundary line is now always visible:
  - `This screen does not place trades.`
- service-owned execution-boundary status text now uses plain-language check/handoff phrasing while preserving blocked/ready/simulated semantics.

## P5-R15 Terminology Preserved

The following user-facing terms remain intact:

- `Entry price`
- `Stop-loss price`
- `Target price`
- `Prepared planning levels`
- `Planning inputs only`
- `Your own values remain authoritative`

Internal contracts and type names may still use `reference` naming where already established.

## What Was Deliberately Not Changed

- no readiness, submission, adapter, or confirmation logic changes
- no risk math or selector changes
- no prepared stop/target selector precedence changes
- no new Trade Hub features or cards
- no live execution, adapter dispatch, broker payload construction, or order submission
- no recommendation/prediction/profit framing

## What Future Work Can Build On

- profile-sensitive copy refinements on the same service-owned semantics
- additional plain-language polish for advanced execution-boundary explainers without widening scope
- continued copy consistency checks across Trade Hub tests and docs

## Verification

- Trade Hub screen labels and card eyebrows updated in app presentation only
- service-owned execution-boundary display copy updated in one shared helper
- placement and execution-boundary view tests updated to lock wording
- no `docs/source` changes and no `CANON` changes
- preview check is required for this phase because visible Trade Hub copy and section hierarchy text changed
