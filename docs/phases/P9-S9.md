# P9-S9 - Strategy Preview Mobile Density / Progressive Disclosure

## Why This Phase Happened Now

`P9-S6` through `P9-S8` strengthened Strategy Navigator semantics (fit-contrast, nearby alternatives, metadata normalization).

That created a predictable UX pressure point: the preview card now carries more useful context, but mobile density can feel crowded.

`P9-S9` happens now to improve scannability and hierarchy without changing strategy logic ownership.

## What Was Changed

P9-S9 adds a conservative presentation-only pass in `app/`:

1. Primary context remains always visible:
   - strategy + scenario header
   - main preview focus
2. Secondary detail becomes progressively disclosed:
   - compact preview detail by default with clear show more/less control
   - compact supporting context by default with clear show more/less control
   - compact optional reading by default with clear show more/less control
3. Fit-contrast remains subordinate in the supporting shelf.
4. Ambiguity wording remains accessible in collapsed supporting context via explicit fit-contrast handling.
5. Knowledge tap-through remains connected to the same topic route.

No strategy fit, contrast, metadata, or nearby-alternative heuristics moved into app logic.

## What Was Deliberately Not Added

P9-S9 does not add:

- execution prompts or dispatch behavior
- ranking/leaderboard surfaces
- prediction framing or certainty theatre
- new strategy engines, new strategy families, or metadata rewrites
- new top-level navigation or tab systems
- inbox/push mechanics

## What Future Presentation Work Can Build On

Future `P9` UX work can build on this pass with:

- targeted section-level microcopy tightening
- adaptive density by profile/surface once explicitly modeled
- lightweight accessibility refinements for disclosure controls

Future work should continue preserving service-owned semantics and keep app ownership limited to presentation behavior.

## Recommendation Review

- Adopt now: conservative progressive disclosure and hierarchy cleanup for mobile Strategy Preview density.
- Add to backlog: profile-aware disclosure defaults once there is explicit product need.
- Decline for now: major navigator redesigns, ranking surfacing, or recommendation-forward interaction patterns.
