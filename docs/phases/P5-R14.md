# P5-R14 - Prepared Reference Limitation Copy / Source Confidence Polishing

## Why This Phase Happened Now

`P5-R13` already introduced one canonical prepared stop/target availability seam and threaded it through plan, preview, confirmation-session, and risk-tool support paths.

`P5-R14` happens now because the seam is stable enough to harden wording posture without widening functionality: source labels, limitation notes, and unavailable wording are now normalized in one service-owned copy seam so downstream consumers remain calm, consistent, and honest.

## What Was Polished

- Added one canonical service-owned copy helper for prepared stop/target references:
  - labels
  - source labels
  - limitation notes
  - unavailable reason wording
- Kept prepared reference output deterministic while normalizing legacy source label variants into canonical copy.
- Preserved one explicit unavailable contract (`NO_STRATEGY_REFERENCE`, `THIN_CONTEXT`, `NOT_ENABLED_FOR_SURFACE`) and added canonical service-owned wording for those reasons without widening execution behavior.
- Added focused tests that lock:
  - canonical source/limitation wording
  - legacy-label normalization
  - thin-context unavailable honesty
  - manual override precedence
  - downstream non-duplication of source/limitation wording logic

## What Was Deliberately Not Added

- no live execution or dispatch behavior
- no broker adapter integration or payload widening
- no multi-target or laddered reward framing
- no account-scoped reference preferences or settings UX
- no quote-assisted predictive stop/target expansion
- no app-owned source-confidence interpretation or wording rules
- no recommendation/profit/certainty theatre

## Preserved P5-R13 Behavior

- explicit prepared values still outrank strategy fallback values for the same stop/target field
- strategy fallback still publishes only when scoped strategy context is concrete enough
- thin or ambiguous context still returns unavailable instead of invented references
- selected-plan preview/session/risk-tool seams still consume one canonical prepared reference producer path
- Trade Hub remains support-first and non-dispatching

## What Future P5 Work Can Build On

- additional deterministic strategy-owned stop/target sources on the same copy seam
- calmer unavailable explanations on future surfaces by reusing the existing service-owned reason-copy helper
- optional richer annotations only if they remain explicit, non-executional, and non-predictive

## Verification

- Added and updated service tests for canonical copy shaping, normalization, precedence preservation, and downstream copy ownership boundaries.
- `npm run verify` remains the merge gate for this phase.

## Recommendation Review

### Adopt now

- one canonical prepared-reference copy seam in `services/trade/`
- canonical unavailable reason wording helper on the same service-owned seam
- downstream consumer pass-through posture (no local source/limitation rewriting)

### Add to backlog

- broader surface rendering of unavailable reason wording where it improves comprehension without adding urgency
- future strategy-specific copy refinements if additional deterministic reference sources are introduced

### Decline for now

- recommendation-style source confidence wording
- predictive/profit framing for prepared references
- execution-path widening, broker payload behavior, or order submission semantics
