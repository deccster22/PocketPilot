# P8-I8 - Export Foundation

## Why This Phase Happened Now

P8-I5 added one calm monthly / quarterly summary foundation.
P8-I6 added one calm summary-archive shelf.
P8-I7 added one calm Year in Review path.

That made the next safe step clear:

- keep building on the same reflection spine rather than starting a new reporting lane
- add one canonical export foundation that can prepare reflection output honestly
- keep export availability, shaping, and limitations in `services/`

P8-I8 happens now because Insights already has enough prepared reflection material to support one explicit export seam, but it is still too early for freeform journaling, AI commentary, full export dispatch workflows, or a broad reporting center.

## What P8-I8 Added

P8-I8 adds five narrow refinements:

1. One canonical export contract family in `services/insights/types.ts` for export formats, option availability, and prepared export requests.
2. One service-owned export options builder in `services/insights/createExportOptionsVM.ts`.
3. One service-owned export preparation seam in `services/insights/createPreparedExportRequest.ts` plus fetch wrappers for the Insights surface.
4. One minimal subordinate `InsightsExportScreen` that renders prepared export options and one prepared export preview only.
5. One light handoff from `InsightsScreen` into the export path without creating a new top-level hub.

The read path is now:

`EventLedger -> createExportOptionsVM / createPreparedExportRequest -> fetchExportOptionsVM / fetchPreparedExportRequest -> InsightsExportScreen`

## Reflection And Export Rules Locked In Here

P8-I8 establishes these rules:

- one canonical export format set only: `PDF_SUMMARY`, `CSV_SUMMARY`, and `CSV_EVENT_LEDGER`
- export option availability stays in `services/`
- prepared export shaping stays in `services/`
- `app/` renders prepared export options and prepared request details only
- export copy stays calm, descriptive, and non-moralizing
- profile access can change visibility, but it does not change underlying event truth
- CSV-oriented exports keep the covered period explicit and keep timezone labeling visible
- provider/runtime diagnostics, raw signal codes, and internal metadata stay out of user-facing export contracts by default
- event-level export widens only when the user explicitly selects the event-ledger format

## What P8-I8 Deliberately Did Not Add

P8-I8 does not add:

- real file dispatch workflows
- a giant export-management center
- journaling or freeform reflection editing
- AI-generated commentary
- compare-window customization sprawl
- gamified analytics
- scorecards or streaks
- push notifications
- background polling
- raw diagnostics export

## What Future Reflection Work Can Build On

Later `P8` work can safely extend this seam with:

- actual file creation or share workflows once a calm dispatch pattern is explicitly scoped
- richer archive-to-export source selection if reflection shelves widen later
- broader annual export shaping once Year in Review grows beyond its first debrief foundation
- optional journal attachment or note-link export once the log / journal family is ready

Those future steps should deepen the same service-owned seam rather than turning Insights into a report center, raw data dump, or export-control dashboard.

## Recommendation Review

- Adopt now: One canonical export foundation because the period summary, summary archive, and Year in Review seams already exist and can now support an honest reflection export path.
- Add to backlog: Real file creation / sharing, richer source selection, and journal-linked export follow-through because they may be useful later but widen scope beyond this foundation.
- Decline for now: Export-management centers, AI commentary, gamified recap flows, and raw diagnostics export because they would turn reflection into reporting theatre instead of a calm support lane.
