# P6-A8 - Since Last Checked Viewed-State Refinement

## Why This Phase Happened Now

P6-A7 established the first calm Since Last Checked section on Snapshot and proved the section could stay compact, account-scoped, and service-owned.

That made the next honest rung clear: refine the same section so it behaves like a front-of-house briefing, not a permanent shelf.

This phase happened now because the section already existed, the event/history foundation was already stable, and the product needed one explicit service-owned clear-after-view seam before any broader P8 reflection work should be considered.

## What Was Refined

- one canonical Since Last Checked display-state seam in `services/orientation/createSinceLastCheckedDisplayState.ts`
- one service-owned snapshot-view acknowledgement seam in `services/orientation/sinceLastCheckedViewedState.ts`
- one canonical fetch path that now returns a prepared display state in `services/orientation/fetchSinceLastCheckedVM.ts`
- one Snapshot surface thread that carries the prepared display state in `services/snapshot/fetchSnapshotSurfaceVM.ts`
- one screen refresh seam that clears the section after a visible view without app-owned ranking or filtering
- one screen-view helper that renders the prepared display state only
- one test path proving the section clears after view and reappears when newer meaningful changes exist

## What Was Deliberately Not Added

- no push notifications
- no background polling
- no inbox or unread model
- no badge counts
- no notification center
- no raw event feed
- no generic history browser
- no manual archive controls
- no urgency language
- no delivery-platform expansion

The section remains calm, compact, foreground-only, and subordinate to Snapshot.

## What Future P6 / P8 Work Can Build On

Later `P6` work can keep tightening copy or eligibility rules for the same calm section without changing ownership.

Later `P8` work can build broader reflection surfaces on the same account-scoped history spine without needing to reinvent viewed-state boundaries.

Future work should preserve the same service-owned posture: explicit inputs, deterministic outputs, and no hidden automation.

## Recommendation Review

### Adopt Now

- Service-owned clear-after-view refinement: adopt now because the section now behaves like a briefing window instead of a permanent shelf.
- Canonical display-state seam: adopt now because the app should consume one prepared surface contract rather than infer visibility locally.

### Add To Backlog

- Additional calm copy refinement once real usage shows where the section feels too terse or too dense
- Broader P8 reflection work that builds on the same spine but belongs in the reflection family, not here

### Decline For Now

- Push, background, inbox, badge, and notification-center behavior
- Manual archive management for Snapshot
- Raw event browsing on Snapshot
